## 1. 서비스 본질 재정의

### 한 줄 정의

**"매일 1분, 시장을 읽는 눈을 키우는 예측 게임"**

시그널플레이는 투자 정보 서비스가 아니다. 투자 뉴스를 소재로 한 **일일 예측 게임**이다. 사용자는 전문가가 되는 것이 아니라, 전문가 5명과 겨루면서 "시장을 보는 감각"을 쌓는 재미를 느낀다.

### 포지셔닝: 투자 앱도 아니고, 게임 앱도 아닌 곳

| 축 | 투자 앱 (토스증권, 삼성POP) | 시그널플레이 | 게임 앱 (야구9단, 퀴즈앱) |
|---|---|---|---|
| 핵심 감정 | 불안, 탐욕 | **"나 꽤 잘 보는데?"** | 재미, 경쟁 |
| 소재 | 실제 매매 | 실제 뉴스 기반 예측 | 가상 시나리오 |
| 리스크 | 돈을 잃음 | 자존심만 잃음 | 없음 |
| 리텐션 드라이버 | 수익률 | 적중률 + 스트릭 | 랭킹 + 보상 |

시그널플레이가 서야 할 자리: **"진짜 시장 뉴스로 하는 예측 게임."** 투자 앱의 소재 진정성 + 게임 앱의 가벼운 참여 구조를 결합한다.

경쟁자는 토스증권이 아니라 **"출근길에 뭐 할까?"** 라는 질문이다. 인스타 스토리 한 번 넘기는 시간에, 시장 질문 하나에 투표하고 전문가 예측을 훑는 경험을 제공한다.

### "방구석 전문가" 세계관 강화

현재 문제: 캐릭터 5명이 있지만, 이름(엑셀형/도서관형 등)이 방법론 설명에 그친다. 사용자가 "내 캐릭터"를 고를 이유가 없다.

**강화 방향: 캐릭터에 성격을 입혀라.**

| 캐릭터 | 현재 | 강화 후 |
|--------|------|---------|
| 엑셀형 (밸류김) | PER/PBR 분석 | "숫자가 거짓말을 하면 내가 고친다." 냉정하고 독설적. 틀리면 "데이터가 부족했을 뿐" |
| 도서관형 (팩터박) | 학술 이론 적용 | "2003년 논문에 이미 답이 있다." 잘난 척 심하지만 근거 탄탄. 틀리면 "시장이 비효율적이었다" |
| 뉴스형 (뉴스최) | 뉴스 센티멘트 | "아까 속보 봤어? 다들 모르더라." 소식통 자랑. 틀리면 "기사 톤이 바뀌었다" |
| 차트형 (봉준선) | 기술적 패턴 | "차트는 모든 것을 말해준다." 신비주의. 틀리면 "노이즈였다" |
| 운형 (코인토) | 감으로 찍기 | "오늘 기분이 좋으니까 호재." 순수. 맞히면 "역시 감이지!" 틀리면 "내일은 맞출 거야" |

**실행 포인트:**
- 각 캐릭터의 reasoning 끝에 **한 줄 코멘트**를 추가한다. "이건 당연한 거 아닌가?" (도서관형) 같은 말투.
- 결과 페이지에서 캐릭터가 맞혔을 때/틀렸을 때 **반응 대사**를 보여준다. "내 모델이 맞았다. 당연하지." vs "...데이터를 재검토하겠다."
- 사용자가 가장 많이 의견이 일치하는 캐릭터를 **"너의 투자 성향"**으로 보여준다. "당신은 팩터박과 70% 일치합니다."

## 2. 핵심 사용자 여정 재설계

### Day 0: 첫 방문 (감정 곡선: 호기심 -> 이해 -> 참여 -> 보상)

**현재 문제:**
- 온보딩 3슬라이드가 기능 설명에 그침 ("투자 질문을 던져요", "전문가가 예측해요", "군중과 비교해요")
- "어제 결과가 나왔어요!" 배너가 첫 방문자에게도 노출 -> ResultPage가 비어있어서 신뢰 파괴

**재설계:**

```
[진입] 토스 앱인토스에서 발견
  |
  v
[0초] 온보딩 없이 바로 오늘의 질문 노출
      "삼성전자 1분기 실적, 주가에 호재일까?"
      -> 질문 자체가 온보딩이다. 설명하지 말고 보여줘라.
  |
  v
[5초] 호재/글쎄/악재 3버튼 탭
      -> 로그인 불필요. 탭 한 번이면 참여 완료.
  |
  v
[6초] 즉시 보상: 군중 비율 공개 + 성공 애니메이션
      "토스 유저 4,821명 중 58%가 호재를 골랐어요"
      -> "나만 다르게 봤네?" 또는 "역시 다들 같은 생각이구나"
  |
  v
[10초] 캐릭터 예측 자동 스크롤 유도
      첫 1명은 펼쳐진 상태로 보여준다 (현재는 전부 접힘)
      -> "이 사람은 왜 악재라고 했지?" 궁금증 유발
  |
  v
[30초] 공유 카드: "너는 어떻게 봐?"
      -> 오늘 할 일 끝. 내일 결과 확인하러 다시 오게 만든다.
```

**핵심 변경:**
1. 첫 방문자에게 온보딩 슬라이드를 보여주지 않는다. 질문 화면이 곧 온보딩이다. 기능 설명은 첫 투표 완료 후 바텀시트로 간결하게.
2. "어제 결과" 배너는 **실제 result_ready 데이터가 있을 때만** 노출한다. 현재 코드는 항상 노출.
3. 캐릭터 카드 첫 번째(적중률 1위)는 기본 펼침 상태로.

### Day 1: 첫 결과 (감정 곡선: 기대 -> 확인 -> 자극)

**현재 문제:**
- 결과 확인의 "아하" 순간이 약하다. 맞혔는지 틀렸는지가 스크롤해야 보임.
- actual_outcome 결정 파이프라인이 미설계 (admin/set-result.ts가 있지만 수동)

**재설계:**

```
[진입] 푸시 알림 또는 자발적 재방문
  |
  v
[0초] 결과 페이지 최상단: 크게 한 줄
      "당신의 예측: 호재 -> 실제 결과: 호재 -- 적중!"
      또는 "당신의 예측: 호재 -> 실제 결과: 악재 -- 빗나감"
      -> 3초 안에 맞혔는지 확인. 스크롤 불필요.
  |
  v
[3초] 나 vs 군중 vs 전문가 3자 비교
      "군중 58%가 호재 -> 군중 적중"
      "엑셀형 적중, 차트형 빗나감"
      -> "나는 엑셀형이랑 같이 맞혔네"
  |
  v
[5초] 내 누적 성적 카드 (신규)
      "3일 연속 적중 | 이번 달 적중률 67% | 엑셀형과 70% 일치"
      -> 다시 올 이유의 핵심. "내 적중률을 깨고 싶다."
  |
  v
[8초] 오늘 투표 유도
      "오늘의 질문이 도착했어요" 버튼
      -> 결과 확인 -> 오늘 투표의 자연스러운 연결
```

**핵심 변경:**
1. ResultPage 최상단에 "내 예측 vs 실제 결과"를 즉시 노출. 현재처럼 질문 카드 안에 묻히면 안 된다.
2. 누적 성적 카드를 추가한다. 현재 voteHistory.ts가 7일만 보관하는데, 이것을 확장.
3. actual_outcome 자동화: 다음 장 마감 후 주가 변동률 기준으로 자동 판정하는 cron을 설계한다 (4장 참조).

### Day 7: 습관화 (핵심 루프가 돌기 시작하는 시점)

**매일 오게 만드는 3가지 장치:**

1. **스트릭**: "5일 연속 참여" -> 끊기 아까움. 투표만 하면 유지됨 (맞힐 필요 없음).
2. **적중률 그래프**: 내 적중률이 올라가는/내려가는 그래프. "이번 주는 60%인데, 저번 주 75%였는데..." 자기 경쟁.
3. **캐릭터 매칭 변화**: "이번 주 당신은 뉴스최와 가장 비슷해요" -> 지난주엔 엑셀형이었는데? 변화 자체가 콘텐츠.

### 이탈 지점과 방어

| 이탈 시점 | 원인 | 방어책 |
|-----------|------|--------|
| Day 0 투표 전 | "이게 뭐하는 앱이지?" | 온보딩 제거, 질문을 바로 보여줌 |
| Day 1 결과 없음 | 결과 페이지가 비어있음 | result_ready 없으면 배너 숨김 + "내일 결과가 나와요" 안내 |
| Day 3 잊어버림 | 다시 올 이유 없음 | 스트릭 시스템 + "어제 군중 60%가 틀렸어요" 푸시 |
| Day 14 질림 | 매일 같은 패턴 | 플래시 투표(속보 기반 긴급 투표) + 주간 리포트 |

## 3. 핵심 루프 강화 방안

### 3-1. 투표 -> 결과 -> 재방문 루프 강화

현재 루프: `질문 읽기 -> 투표 -> (24시간 대기) -> 결과 확인 -> 끝`

이 루프의 문제는 **결과 확인 후 할 일이 없다**는 점이다.

**강화된 루프:**

```
질문 읽기 -> 투표 -> 군중 비율 즉시 확인 -> 캐릭터 예측 비교
     ^                                              |
     |                                              v
오늘 투표 <-- 결과 확인 <-- (다음날) <-- 내 성적 업데이트
                |
                v
         공유 ("나 3일 연속 적중!")
```

**구체적 강화 포인트:**

1. **투표 직후 인게이지먼트 확대**: 투표하면 군중 비율이 나오는데, 여기서 끝나지 않고 "전문가 5명 중 3명이 당신과 같은 선택을 했어요"를 보여준다. 사용자의 선택을 캐릭터와 즉시 연결.

2. **결과 페이지에서 "다음 액션" 명확화**: 결과 확인 후 바로 오늘 질문으로 연결. "어제 틀렸으니 오늘은 신중하게" 같은 맥락 연결 문구.

3. **주간 리포트**: 일요일에 "이번 주 당신의 성적: 5/7 적중, 엑셀형보다 나았어요!" 자동 생성.

### 3-2. 사용자 성과 축적 시스템

| 요소 | 설계 | 왜 이게 작동하는가 |
|------|------|-------------------|
| **투표 스트릭** | 연속 투표 일수. 1회라도 빠지면 리셋. | 손실 회피. "7일 스트릭 깨기 아까워서라도 투표" |
| **적중률** | 최근 30일 기준 적중/총투표. 백분율 표시. | 자기 경쟁. "지난달 62%였는데 이번 달은..." |
| **레벨** | 누적 투표 수 기반. 10회=루키, 30회=레귤러, 100회=프로, 365회=마스터 | 장기 리텐션. 레벨 표시가 공유 카드에 노출 |
| **배지** | 특정 조건 달성. "5연속 적중", "군중과 반대로 맞힘", "운형과 3연속 일치" | 수집욕 + 바이럴 (희귀 배지 자랑) |
| **캐릭터 싱크로율** | 각 캐릭터와 예측 일치율. "당신은 뉴스최와 72% 싱크로" | 정체성 부여. "나는 뉴스파야" |

**레벨 시스템 상세:**

```
Lv.1  시장 구경꾼    (1~9회 투표)
Lv.2  초보 분석가    (10~29회)
Lv.3  시장 감시자    (30~99회)
Lv.4  프로 예측가    (100~364회)
Lv.5  시그널 마스터  (365회+)
```

레벨은 공유 카드와 리더보드에 표시된다. "프로 예측가가 호재를 골랐습니다"는 "시장 구경꾼이 호재를 골랐습니다"보다 무게감이 다르다.

### 3-3. "나 vs 캐릭터 vs 군중" 3각 경쟁

현재는 투표 후 군중 비율만 보여주고, 캐릭터 예측은 별개로 나열된다. 이 셋을 **하나의 경쟁 구도**로 묶어라.

**결과 페이지 3자 비교 카드:**

```
이번 달 적중률 순위
1위  엑셀형 (밸류김)   72%   ████████░░
2위  나               67%   ███████░░░
3위  군중             61%   ██████░░░░
4위  뉴스형 (뉴스최)   60%   ██████░░░░
5위  도서관형 (팩터박)  58%   █████░░░░░
...
```

사용자가 캐릭터보다 높으면: "당신이 엑셀형을 이겼습니다!" -> 공유 유발.
사용자가 낮으면: "엑셀형이 3개월 연속 1위입니다. 이길 수 있을까요?" -> 재방문 유발.

**핵심: 군중 적중률도 추적한다.** "군중이 이번 달 61% 적중"이면, 사용자에게 "군중보다 잘하고 있다"는 피드백을 준다. "군중보다 못하면 그냥 다수결을 따르는 게 낫다"는 도발적 메시지도 가능.

## 4. 콘텐츠 볼륨 문제 해결

### 현재 상태 진단

- 하루 1개 질문 (api/question.ts에서 Gemini로 1개 생성)
- 캐릭터 예측 5개 (전부 접혀있어서 사실상 0개)
- 이번 주 추천픽 섹션 (WeeklyPicksSection, lazy load)
- 세션 시간: 투표 10초 + 결과 확인 10초 = 약 20-30초

### 해결 전략: "하루 1질문"은 유지하되, 소비할 콘텐츠를 늘려라

질문 수를 늘리면 투표의 희소성이 사라진다. "하루 1번 기회"가 긴장감의 원천이다. 대신 투표 전후에 읽을 거리를 만든다.

**레이어 1: 캐릭터 예측을 제대로 소비시켜라 (현재 자산 활용)**

현재 CharacterCard가 전부 접혀있어서 핵심 콘텐츠가 도달하지 않는다.

- 적중률 1위 캐릭터는 기본 펼침
- 나머지 4명은 접힘이되, 헤더에 reasoning 첫 줄 미리보기 표시 (현재 이름+호재/악재 뱃지만 보임)
- "의견이 갈리는 포인트" 하이라이트: "엑셀형은 호재, 차트형은 악재 -- 무엇이 다른가?"를 자동 생성

**레이어 2: 오늘의 시그널 해설 (신규)**

투표 마감 후 ~ 결과 발표 전 시간대에 "오늘의 시그널 해설" 콘텐츠를 생성한다.

- Gemini가 실제 장중 뉴스를 검색해서 "오늘 질문과 관련된 움직임" 요약
- "장 시작 후 삼성전자 +1.2%, 외국인 순매수 200억" 같은 실시간 스냅샷
- 사용자는 투표 후에도 "내 예측이 맞아가고 있나?" 확인하러 재방문

**레이어 3: 플래시 투표 (속보 기반, P3 백로그에 이미 있음)**

정규 질문 외에, 속보 발생 시 1~2시간 한정 긴급 투표를 연다.

- "속보: 미국 CPI 예상치 상회 -- 코스피 어떻게 될까?" (2시간 한정)
- 하루 0~1회. 속보가 없으면 없는 날도 있다 (희소성 유지).
- 정규 투표와 별개로 적중률 집계. "플래시 적중왕" 별도 배지.

**레이어 4: 주간 리포트 (일요일 콘텐츠)**

- 이번 주 질문 5개 요약 + 각 결과
- 캐릭터별 주간 적중률 변동
- "이번 주 MVP: 뉴스최 (5/5 적중)"
- 사용자 개인 주간 성적 카드

### 세션 시간 1분 -> 5분 달성 경로

| 콘텐츠 | 예상 소비 시간 | 비고 |
|--------|-------------|------|
| 질문 읽기 + 투표 | 15초 | 현재와 동일 |
| 군중 비율 확인 | 10초 | 현재와 동일 |
| 캐릭터 예측 읽기 (1위 펼침 + 2명 추가 클릭) | 60초 | 현재 0초 -> 핵심 개선 |
| "의견 갈림 포인트" 읽기 | 30초 | 신규 |
| 내 성적 카드 확인 | 20초 | 신규 |
| 이번 주 추천픽 스크롤 | 30초 | 현재 존재 |
| 공유 카드 생성/전송 | 30초 | 현재 존재 |
| **합계** | **약 3분 15초** | |

여기에 결과 페이지 재방문 시 2분 추가하면 일일 총 5분 달성 가능.

## 5. 바이럴/공유 전략

### 현재 공유 카드의 문제

share.ts를 보면 순수 텍스트 기반이다:

```
시그널플레이 -- 오늘의 질문
삼성전자 1분기 실적 발표
이번 분기 실적, 삼성전자 주가에 호재일까?
토스 유저 4,821명 투표: 호재 58% | 글쎄 15% | 악재 27%
너는 어떻게 봐?
```

카카오톡이나 메시지에서 이 텍스트를 받으면 **클릭할 이유가 없다**. 링크 미리보기(OG 이미지)도 generic할 가능성이 높다.

### 공유 카드 이미지화 방향

**원칙: 공유 카드는 "결과"를 자랑하는 것이다. 질문을 공유하는 것이 아니다.**

사람들이 공유하는 건 "나 맞혔다!"이지 "이 질문 풀어봐"가 아니다.

**공유 시나리오 3가지:**

1. **투표 직후 공유** (현재 존재, 개선 필요)
   - "삼성전자 실적, 나는 호재로 봤다. 군중 58%도 호재. 너는?"
   - 이미지 카드: 질문 + 군중 비율 바 차트 + 캐릭터 5명 예측 아이콘 한 줄
   - CTA: "나도 투표하기"

2. **결과 확인 후 공유** (핵심 바이럴 포인트)
   - "3일 연속 적중! 이번 달 적중률 67% -- 엑셀형보다 높다"
   - 이미지 카드: 적중 뱃지 + 스트릭 + 적중률 그래프 미니 버전
   - CTA: "나도 도전하기"

3. **주간 리포트 공유** (주 1회)
   - "이번 주 5/5 전승! 시그널 마스터 등극"
   - 이미지 카드: 주간 성적표 형태

**이미지 생성 방식:** 서버사이드에서 html2canvas 또는 Satori(Vercel OG Image) 활용. 클라이언트에서 canvas로 그리는 것은 WebView 호환 이슈가 있으므로 서버에서 PNG를 생성해 URL로 제공.

### 토스 내 자연 발견 경로

앱인토스 플랫폼 특성을 활용한다:

1. **토스 피드**: 앱인토스 서비스는 토스 메인 피드에 노출될 수 있다. "오늘 4,821명이 투표한 질문"을 피드 카드로.
2. **토스 증권 연계**: 토스 증권에서 종목 상세 볼 때 "이 종목 관련 시그널플레이 질문이 있어요" 딥링크 (앱인토스 API 지원 범위 내에서).
3. **뱃지/프로필**: 토스 프로필에 시그널플레이 뱃지 표시 (Bedrock SDK 활용 가능성 검토 필요).

### 입소문 유발 시나리오

**"언제 친구에게 보내고 싶어지는가?"**

1. **내가 맞혔을 때**: "야 나 삼성전자 호재 맞혔다 ㅋㅋ 엑셀형이랑 같이" -> 자랑
2. **군중이 틀렸을 때**: "4,000명이 호재 골랐는데 악재였대 ㅋㅋ 나만 맞힘" -> 소수파 자랑
3. **캐릭터가 웃길 때**: "운형이 '점심에 짜장면 먹었으니 호재'래 ㅋㅋㅋ" -> 유머 공유
4. **질문이 논쟁적일 때**: "미국 금리 동결인데 너는 호재로 봐 악재로 봐?" -> 의견 묻기

3번과 4번이 현재 가장 쉽게 만들 수 있는 바이럴 포인트다. 운형의 reasoning을 더 웃기게 만들고, 질문 자체의 논쟁성을 Gemini 프롬프트에서 강화한다.

## 7. 성공 지표 (North Star)

### North Star Metric: **일일 투표 완료율**

DAU 중 실제로 투표까지 완료한 비율. 이 하나의 숫자가 올라가면 나머지가 따라온다.

- 투표 완료 = 콘텐츠 소비 (캐릭터 예측 노출)
- 투표 완료 = 다음날 결과 확인 재방문 동기
- 투표 완료 = 공유 카드 생성 가능 (바이럴)
- 투표 완료 = 적중률 축적 (장기 리텐션)

### 지표 체계

| 구분 | 지표 | 현재 추정 | 3개월 목표 | 6개월 목표 |
|------|------|----------|-----------|-----------|
| **활성** | DAU | 0 (미출시) | 500 | 2,000 |
| **참여** | 일일 투표 완료율 | - | 70% (DAU 대비) | 80% |
| **리텐션** | D1 리텐션 | - | 40% | 50% |
| **리텐션** | D7 리텐션 | - | 20% | 30% |
| **깊이** | 평균 세션 시간 | ~30초 | 2분 | 4분 |
| **바이럴** | 공유율 (투표 완료 대비) | ~1% (추정) | 10% | 20% |
| **바이럴** | K-factor (공유 -> 신규 유저) | - | 0.1 | 0.3 |
| **축적** | 7일 스트릭 유지율 | - | 15% | 25% |

### 6개월 후 "성공" 상태

**양적 기준:**
- DAU 2,000명이 매일 투표하고, 그 중 20%가 공유 카드를 보내고, D7 리텐션 30%를 달성한 상태.
- actual_outcome이 자동으로 판정되고, 캐릭터 적중률이 실제 데이터로 운영되는 상태.

**질적 기준:**
- "시그널플레이 해봤어?"가 토스 투자 커뮤니티에서 가끔 언급되는 상태.
- 사용자가 "내 적중률 67%"를 자랑스럽게 공유하는 상태.
- "엑셀형이 이번 달 또 1등이래"가 하나의 밈이 되는 상태.
- 방구석 전문가 캐릭터 중 최소 1명이 "팬"을 가지는 상태 ("나는 팩터박파야").

**이 상태가 되면 다음 단계로:**
- AdMob 수익화 시작 (DAU 2,000 이상에서)
- 토스 로그인 연동 (크로스 디바이스 데이터 보존)
- 플래시 투표 + 관심 종목 (콘텐츠 확장)

## 부록: actual_outcome 자동 판정 파이프라인 설계

현재 가장 큰 운영 리스크. admin/set-result.ts로 수동 판정하는 구조인데, 매일 사람이 판정하면 서비스가 돌아가지 않는다.

### 판정 기준

| 카테고리 | 기준 | 호재 | 악재 | 글쎄 |
|----------|------|------|------|------|
| 종목 | 해당 종목 당일 종가 변동률 | +1% 이상 | -1% 이상 | -1% ~ +1% |
| 지수 | 코스피/코스닥 당일 종가 변동률 | +0.5% 이상 | -0.5% 이상 | -0.5% ~ +0.5% |
| 매크로 | Gemini 판정 (뉴스 기반) | AI 판정 | AI 판정 | AI 판정 |

### 파이프라인

```
[매일 16:00 KST] Vercel Cron 실행
  |
  v
[1] 어제 질문의 category 확인
  |
  v
[2-A] 종목/지수: 금융 API(한국투자증권 OpenAPI 등)로 종가 조회 -> 변동률 계산 -> 자동 판정
[2-B] 매크로: Gemini에게 "어제 이 질문의 실제 결과가 어떻게 됐는지" 판정 요청 (Google Search Grounding)
  |
  v
[3] actual_outcome 업데이트 + result_ready = true
  |
  v
[4] AI 총평 생성 (Gemini): "군중은 맞혔고, 차트형은 틀렸다. 그 이유는..."
```

매크로 카테고리의 AI 판정은 정확도가 낮을 수 있다. 초기에는 **종목/지수 카테고리 질문 비율을 80% 이상으로 유지**해서 자동 판정의 신뢰도를 확보한다.
