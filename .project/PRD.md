## 1. 배경 & 문제 정의

### 배경

토스 앱 사용자의 대다수는 주식·ETF 계좌를 보유하고 있지만, 매일 경제 뉴스를 챙겨보는 사람은 소수다. 경제 동향 파악의 니즈는 높지만 진입 장벽(양이 많음, 어려운 용어, 재미없음)이 실제 행동을 막는다.

### 핵심 문제

1. **정보 과부하**: 경제 뉴스는 하루에도 수백 건. 오늘 투자자가 꼭 알아야 할 것만 골라내는 큐레이션이 없다.
2. **낮은 지속성**: 한두 번 경제 뉴스를 읽어도 습관으로 이어지지 않는다. 리마인더나 재미 요소 없이는 매일 접속할 이유가 없다.
3. **소극적 정보 소비**: 뉴스를 읽어도 "이게 내 투자에 좋은 건지 나쁜 건지" 판단하기 어렵다. 수동적 소비에 그친다.
4. **진입 장벽**: 별도 앱 설치·가입이 필요한 서비스는 이탈률이 높다.

### 솔루션 가설

토스 앱 안에서 **매일 3분**, AI가 요약한 경제 뉴스 3개를 카드로 보여주고, 각 뉴스가 **호재/악재/영향없음**인지 예측하게 하면:
- 정보 소비 진입 장벽이 낮아지고 (30초에 오늘 동향 파악)
- 예측 결과를 다음날 확인하는 루프가 매일 접속 습관을 만들며
- 게임화된 리더보드가 지속적 참여를 유도한다.

## 2. 타겟 사용자

### Primary: 토스 앱 사용 2030대 직장인

- **연령**: 25~39세
- **특성**: 토스 앱을 주 금융 앱으로 사용. 투자 계좌 보유. 게임·챌린지에 친숙.
- **페인포인트**: 경제 뉴스 팔로업은 하고 싶지만 바빠서 못 한다. 어떤 뉴스가 중요한지 모르겠다.
- **사용 시나리오**: 출퇴근 중 3분, 토스 앱에서 미니앱 탭 → 오늘의 시그널 3개 확인 + 예측 → 다음날 결과 확인.

### Secondary: 경제 공부 중인 20대 대학생/취준생

- **특성**: 투자 초보, 경제 용어 학습 목적으로도 사용.
- **사용 시나리오**: AI 요약 카드로 경제 용어와 흐름을 자연스럽게 학습.

### Anti-persona: 전문 트레이더 / 주식 데이투레이더

- **이유**: 당사 시그널의 데이터 깊이와 실시간성이 그들의 요구를 충족하지 못함. 이 그룹을 위한 설계는 핵심 타겟 경험을 복잡하게 만든다.

## 3. 핵심 기능 명세 (3개 화면)

### 3.1 홈 — 오늘의 시그널 (`/`)

- **시그널 카드 3개**: Gemini AI가 매일 오전 8시에 경제 뉴스를 분석해 생성. 각 카드는 제목(30자 이내) + 요약(80자 이내) + 카테고리(경제지표/기업동향/글로벌이슈) 포함.
- **예측 버튼**: 각 카드에 호재 / 악재 / 영향없음 3개 버튼. 당일 1회 제출, 수정 불가.
- **제출 상태 표시**: 예측 완료된 카드는 선택 내용 표시 + 비활성화.
- **면책 문구**: 화면 하단 고정 "본 콘텐츠는 투자 참고 정보이며, 투자 책임은 본인에게 있습니다."

### 3.2 결과 — 어제의 시그널 결과 (`/result`)

- **결과 카드**: 어제 예측한 시그널 3개의 실제 결과(호재/악재/영향없음) vs 내 예측 비교.
- **점수 표시**: 맞은 개수 / 3 + 획득 점수(맞으면 +10, 스트릭 보너스 +5×연속 일수).
- **스트릭 표시**: 연속 예측 참여 일수 (예측 미제출 시 스트릭 초기화).
- **광고 (리워드)**: "결과 보기" 탭 시 선택적 리워드 광고 노출 (Bedrock AdMob SDK). 시청 시 보너스 점수 +5.
- **공유 유도**: 결과 카드 이미지로 공유 버튼 (토스 공유 시트 사용).

### 3.3 리더보드 (`/leaderboard`)

- **주간 랭킹**: 토스 리더보드 SDK 연동. 이번 주 총 점수 기준 상위 랭킹 표시.
- **내 순위**: 현재 나의 주간 순위 + 점수 표시.
- **랭킹 갱신**: 매일 자정 점수 집계 후 리더보드 업데이트.
- **광고**: 리더보드 진입 시 전면 광고 1회/일 노출 (Bedrock AdMob SDK).

## 7. 앱인토스 규정 준수

### 필수 준수 항목

- **투자 자문 금지**: 모든 시그널 카드에 "투자 참고 정보" 레이블 표시. 홈/결과 화면 하단 면책 문구 고정 노출.
- **금융상품 판매 금지**: 특정 증권사, 펀드, ETF 상품을 추천하거나 가입 링크를 제공하는 콘텐츠 금지.
- **암호화폐 직접 언급 금지**: 비트코인, 이더리움, NFT 등 가상자산을 시그널 주제로 직접 다루지 않음. Gemini 프롬프트에 명시적으로 제외 지시 포함.
- **게임/도박법**: 예측 게임 결과에 현금·포인트 환전·실물 보상 연결 금지. 리더보드 순위와 점수(게임 내)만 보상으로 제공.
- **광고 표기**: AdMob 광고 영역에 "광고" 레이블 명시. 네이티브 광고 형식은 토스 광고 가이드라인 준수.

### Gemini 프롬프트 가이드라인

시그널 생성 프롬프트에 다음 제약 조건 반드시 포함:
```
- 암호화폐, 가상자산 관련 뉴스는 시그널에 포함하지 마세요.
- 특정 종목, 특정 금융상품 구매/판매를 권유하는 표현을 사용하지 마세요.
- "투자하세요", "매수하세요" 등 직접적인 투자 권유 문구 금지.
- 경제 동향, 거시 지표, 기업 실적 등 사실 기반 정보로만 구성하세요.
```

## 8. 리스크 & 가정

### 규제 리스크

- **투자 자문 해석**: AI 시그널이 투자 자문으로 해석될 경우 → 면책 문구 + "AI 생성 콘텐츠" 명시 + 직접 권유 표현 금지로 대응.
- **앱인토스 심사**: 토스 미니앱 심사에서 금융 관련 콘텐츠 규정 위반 판정 위험 → 위 규정 준수 항목 사전 확인 후 제출.

### 기술 가정

- Bedrock SDK가 WebView SPA(Vite+React) 환경에서 정상 동작한다.
- Gemini API 무료 티어(분당 15회 요청)로 일 3개 시그널 생성이 충분하다.
- Vercel KV의 무료 티어(30MB, 30,000 req/월)로 MVP 트래픽을 감당할 수 있다.

### 데이터 리스크

- **Gemini 환각(Hallucination)**: 부정확한 정보 생성 가능 → 시그널 카드에 "AI 생성 콘텐츠" 명시 + 원문 소스 링크 의무화.
- **뉴스 소스 품질**: RSS 피드 뉴스 품질에 의존 → 신뢰 출처 화이트리스트 관리.
