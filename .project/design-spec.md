# Design Spec: 시그널플레이 (SignalPlay)

> **소유자**: Designer 피카
> **최종 수정**: 2026-03-22
> **상태**: 초안 (Draft) — 앱인토스 피벗 반영

---

## 1. 디자인 방향

### 컨셉: "토스 안에서 자연스럽게"

시그널플레이는 토스 앱 내 미니앱(앱인토스)으로 동작한다. 별도의 강한 브랜드 정체성보다 토스 앱과 시각적으로 조화롭고, 사용자가 낯설지 않게 느끼는 것이 최우선이다. TDS(토스 디자인 시스템) 기반 컴포넌트를 최대한 활용한다.

**3가지 핵심 감성 키워드**:
1. **Simple** — 복잡한 데이터를 카드 3개로 단순화. 30초에 파악 가능.
2. **Familiar** — 토스 앱 사용자에게 익숙한 화이트/클린 디자인. 학습 비용 0.
3. **Rewarding** — 예측 성공 시 토스 특유의 명확하고 즐거운 피드백.

### 라이트 모드 우선

앱인토스는 토스 앱의 시스템 설정을 따른다. 기본은 라이트 모드. 다크 모드는 시스템 테마 감지(`prefers-color-scheme`)로 Phase 2에서 지원.

## 2. 색상 토큰

TDS 색상 토큰을 우선 사용. 커스텀 토큰은 최소화.

### 기본 팔레트

| 토큰 | 색상 코드 | 용도 |
|---|---|---|
| `--tds-color-primary` | `#3182F6` | 토스 블루 — 브랜드 메인, CTA 버튼 |
| `--tds-color-primary-hover` | `#1B6FE4` | 버튼 hover/active |
| `--tds-color-bg-base` | `#FFFFFF` | 기본 배경 |
| `--tds-color-bg-surface` | `#F9FAFB` | 카드, 섹션 배경 |
| `--tds-color-text-primary` | `#191F28` | 본문 텍스트 |
| `--tds-color-text-secondary` | `#6B7684` | 보조 텍스트, 레이블 |
| `--tds-color-text-tertiary` | `#B0B8C1` | 비활성, 힌트 |
| `--tds-color-border` | `#E5E8EB` | 카드 테두리, 디바이더 |
| `--tds-color-success` | `#00C471` | 호재 / 정답 / 긍정 |
| `--tds-color-danger` | `#F04452` | 악재 / 오답 / 경고 |
| `--tds-color-warning` | `#FF9500` | 영향없음 / 중립 |
| `--tds-color-overlay` | `rgba(0,0,0,0.4)` | 모달 오버레이 |

### 예측 버튼 색상

| 예측 값 | 기본 배경 | 선택됨 배경 | 텍스트 | 아이콘 |
|---|---|---|---|---|
| 호재 | `#F0FFF7` | `#00C471` | `#00C471` → `#FFFFFF` | ↑ |
| 악재 | `#FFF2F3` | `#F04452` | `#F04452` → `#FFFFFF` | ↓ |
| 영향없음 | `#FFF8EC` | `#FF9500` | `#FF9500` → `#FFFFFF` | → |

## 3. 타이포그래피

### 폰트

| 역할 | 폰트 | 비고 |
|---|---|---|
| 전체 UI | **Pretendard** | 한국어 최적화, TDS 기본 폰트와 동일 계열 |
| 점수/숫자 강조 | **Pretendard** (tabular-nums) | `font-variant-numeric: tabular-nums` |

### 스케일

| 토큰 | 크기 | 두께 | 용도 |
|---|---|---|---|
| `text-xs` | 11px | 400 | 타임스탬프, 면책 문구 |
| `text-sm` | 13px | 400 | 보조 텍스트, 카테고리 칩 |
| `text-base` | 15px | 400 | 시그널 요약 본문 |
| `text-lg` | 17px | 600 | 시그널 제목, 섹션 헤더 |
| `text-xl` | 20px | 700 | 페이지 타이틀 |
| `text-2xl` | 24px | 700 | 점수 숫자, 스트릭 강조 |
| `text-3xl` | 30px | 800 | 결과 화면 핵심 수치 |

## 4. 화면 상세 스펙

### 4.1 홈 — 오늘의 시그널 (`/`)

**레이아웃 구조**
```
┌─────────────────────────────────────┐
│  시그널플레이           [오늘 날짜]  │  ← 상단 헤더 (44px)
├─────────────────────────────────────┤
│                                     │
│  오늘의 시그널 카드 #1               │  ← SignalCard
│                                     │
│  오늘의 시그널 카드 #2               │  ← SignalCard
│                                     │
│  오늘의 시그널 카드 #3               │  ← SignalCard
│                                     │
├─────────────────────────────────────┤
│  본 콘텐츠는 투자 참고 정보입니다.   │  ← DisclaimerBanner (고정)
└─────────────────────────────────────┘
        [홈]    [결과]    [랭킹]         ← BottomNav (TDS)
```

**SignalCard 컴포넌트**
```
┌─────────────────────────────────────┐
│ [카테고리 칩]                        │  ← TDS Chip (sm)
│                                     │
│ 시그널 제목 (최대 2줄)               │  ← text-lg, bold, #191F28
│                                     │
│ AI 요약 텍스트 (최대 2줄)            │  ← text-base, #6B7684
│                                     │
│ 출처 링크 ↗                          │  ← text-sm, 토스 블루
│                                     │
│  [↑ 호재]  [→ 영향없음]  [↓ 악재]   │  ← 예측 버튼 3개
└─────────────────────────────────────┘
```

- **크기**: 전체 폭 (좌우 패딩 16px) × 가변 높이
- **배경**: `--tds-color-bg-surface` (`#F9FAFB`)
- **테두리**: 1px `--tds-color-border`, radius 12px
- **간격**: 카드 사이 12px
- **상태 — 예측 완료**: 선택한 버튼 강조 색, 나머지 버튼 비활성. 카드 전체 불투명도 0.8.
- **스켈레톤**: 로딩 중 제목/요약/버튼 영역을 회색 shimmer로 대체

### 4.2 결과 — 어제의 시그널 결과 (`/result`)

**레이아웃 구조**
```
┌─────────────────────────────────────┐
│  어제의 결과           [날짜]        │  ← 상단 헤더
├─────────────────────────────────────┤
│                                     │
│  ✓ 2 / 3 맞췄어요!    +25점         │  ← ScoreSummary
│  🔥 7일 연속 참여 중                │  ← 스트릭 배지
│                                     │
│ [광고 보고 +5점 받기]                │  ← 리워드 광고 CTA (TDS 버튼 outline)
│                                     │
├─────────────────────────────────────┤
│  결과 카드 #1  ✓ 호재  내예측: 호재  │  ← ResultCard (정답)
│  결과 카드 #2  ✗ 호재  내예측: 악재  │  ← ResultCard (오답)
│  결과 카드 #3  ✓ 호재  내예측: 호재  │  ← ResultCard (정답)
│                                     │
│  [오늘 예측하러 가기 →]              │  ← TDS 버튼 primary
├─────────────────────────────────────┤
│  본 콘텐츠는 투자 참고 정보입니다.   │
└─────────────────────────────────────┘
        [홈]    [결과]    [랭킹]
```

**ResultCard 컴포넌트**
```
┌─────────────────────────────────────┐
│ [카테고리]            [✓ 정답 / ✗ 오답]│
│ 시그널 제목                          │
│                                     │
│ 실제 결과: [호재 ↑]                  │  ← 색상 강조
│ 내 예측:   [호재 ↑]                  │  ← 동일 색 = 정답, 다른 색 = 오답
└─────────────────────────────────────┘
```

- 정답: 상단에 `#00C471` 좌측 보더 4px + 배경 `#F0FFF7`
- 오답: 상단에 `#F04452` 좌측 보더 4px + 배경 `#FFF2F3`

### 4.3 리더보드 (`/leaderboard`)

**레이아웃 구조**
```
┌─────────────────────────────────────┐
│  랭킹                                │  ← 상단 헤더
│           이번 주 TOP 랭킹           │
├─────────────────────────────────────┤
│  🥇 1위   닉네임   245점            │
│  🥈 2위   닉네임   230점            │
│  🥉 3위   닉네임   215점            │
│  ─────────────────────────────────  │
│  4위   닉네임   200점               │
│  5위   닉네임   195점               │
│  ...                                │
├─────────────────────────────────────┤
│  내 순위: 12위   175점   (고정)      │  ← 토스 블루 배경 sticky 행
└─────────────────────────────────────┘
        [홈]    [결과]    [랭킹]
```

- 토스 리더보드 SDK(`bedrock.leaderboard`) 컴포넌트를 래핑. TDS 스타일 적용.
- 1~3위: 메달 이모지 + 미묘한 배경 그라데이션 (금/은/동 톤).
- 내 행: 토스 블루 배경 tint(`#EEF5FE`), 하단 sticky.

## 5. 공통 컴포넌트 스펙

### DisclaimerBanner

- 위치: 홈/결과 화면 BottomNav 바로 위 고정
- 텍스트: "본 콘텐츠는 투자 참고 정보이며, 투자 책임은 본인에게 있습니다."
- 스타일: `text-xs`, `#B0B8C1`, 배경 `#FFFFFF`, 상단 1px 보더 `#E5E8EB`

### BottomNav

- TDS BottomNavigation 컴포넌트 사용
- 탭 3개: 홈(홈 아이콘) / 결과(체크 아이콘) / 랭킹(트로피 아이콘)
- 활성 탭: 토스 블루 `#3182F6`

### 카테고리 칩

- TDS Chip (sm 사이즈)
- 경제지표: 배경 `#EEF5FE`, 텍스트 `#3182F6`
- 기업동향: 배경 `#FFF3E0`, 텍스트 `#E65100`
- 글로벌이슈: 배경 `#F3E5F5`, 텍스트 `#7B1FA2`

## 6. 접근성

### 기준

- **WCAG 2.1 AA** 준수 목표
- 색상 대비: 텍스트 최소 **4.5:1** (소형), 대형 텍스트 3:1
- 예측 버튼(호재/악재/영향없음)은 색상 외 텍스트+아이콘으로 구분 (색맹 대응)

### 구현 체크리스트

- [ ] 예측 버튼에 `aria-label="호재 예측"` / `aria-pressed` 상태 표시
- [ ] 정답/오답 ResultCard에 시각 외 `aria-label="정답"` / `aria-label="오답"` 포함
- [ ] 스켈레톤 로딩 시 `aria-busy="true"` 설정
- [ ] BottomNav 탭에 `aria-current="page"` 활성 탭 표시
- [ ] 탭 영역 최소 44×44px
- [ ] `prefers-reduced-motion` 감지 → 전환 애니메이션 최소화

## 8. 디자인 QA 체크리스트

### 출시 전 필수 확인

**레이아웃**
- [ ] 375px (iPhone SE), 390px (iPhone 15), 412px (Galaxy S24) 세 해상도 확인
- [ ] Safe area (노치, 홈 인디케이터) 침범 없음 — `env(safe-area-inset-*)` 적용 확인
- [ ] 스크롤 시 BottomNav/DisclaimerBanner와 콘텐츠 겹침 없음
- [ ] 토스 앱 상태바(StatusBar) 영역 침범 없음

**색상 & 대비**
- [ ] 모든 텍스트 WCAG AA 색상 대비 통과
- [ ] 예측 버튼을 색맹 필터(Deuteranopia)로 확인 — 텍스트+아이콘으로 구분 가능한지

**타이포그래피**
- [ ] 긴 텍스트 말줄임(`text-overflow: ellipsis`) 정상 작동
- [ ] 시그널 제목 2줄 초과 시 말줄임 처리

**인터랙션**
- [ ] 예측 버튼 선택 → 즉각 색상 피드백 (< 100ms)
- [ ] 예측 제출 완료 후 버튼 비활성화 상태 확인
- [ ] 로딩 스켈레톤 모든 카드에 구현
- [ ] 에러 상태 UI (빈 피드, 네트워크 오류, 결과 집계 중) 구현
- [ ] 리워드 광고 버튼 탭 → Bedrock SDK 광고 호출 정상 동작 확인
