## 2. E2E 시나리오

### 시나리오 1: 회원가입 및 첫 로그인

```
1. 앱 진입 (/)
2. "카카오로 시작하기" 탭
3. 카카오 OAuth 인증 (MSW 모킹)
4. 닉네임 설정 화면
5. 닉네임 입력 → 확인
6. 홈 화면 도달 확인
7. 상단에 닉네임 표시 확인
```

**합격 기준**: 회원가입 완료 후 홈 화면에서 닉네임이 정상 표시된다.

### 시나리오 2: 오늘의 시그널 확인

```
1. 로그인된 상태로 홈 진입
2. 시그널 카드 3~5개 렌더링 확인
3. 각 카드에 섹터 칩, 임팩트 배지, 요약 텍스트 표시 확인
4. 카드 탭 → 시그널 상세 모달 오픈
5. 마감 시간 표시 확인
```

**합격 기준**: 시그널 카드가 로드되고 상세 모달이 정상 오픈된다.

### 시나리오 3: 예측 참여

```
1. 시그널 상세 모달에서 예측 슬라이더 확인
2. "상승" 선택
3. 신뢰도 3단계 선택
4. "예측 제출" 버튼 탭
5. 제출 확인 애니메이션 재생 확인
6. 카드가 "예측 완료" 상태로 변경 확인
7. 동일 시그널 재제출 불가 확인 (중복 제출 방지)
```

**합격 기준**: 예측이 정상 제출되고 중복 제출이 차단된다.

### 시나리오 4: 결과 확인

```
1. 전일 시그널 결과 공개 상태로 앱 진입
2. 홈 피드에서 "결과 공개" 배지 확인
3. 해당 카드 탭 → 결과 모달
4. 내 예측 방향 vs 실제 방향 비교 UI 확인
5. 획득 점수 표시 확인
6. "정답" 시 초록색 피드백, "오답" 시 빨간색 피드백 확인
```

**합격 기준**: 결과 화면에서 예측 결과와 점수가 정확히 표시된다.

### 시나리오 5: 리더보드 확인

```
1. 하단 탭 → 리더보드 이동
2. "주간" 탭 선택
3. 상위 10명 목록 렌더링 확인
4. 내 순위가 하단에 고정 표시 확인
5. 1~3위 금/은/동 아이콘 확인
6. "월간" 탭 전환 확인
```

**합격 기준**: 리더보드가 정상 표시되고 내 순위가 고정 표시된다.

## 3. 엣지 케이스

### 3.1 네트워크 끊김

| 상황 | 기대 동작 |
|---|---|
| 시그널 피드 로드 중 오프라인 | Skeleton 유지 → 에러 토스트 "네트워크 연결을 확인해주세요" |
| 예측 제출 중 오프라인 | 낙관적 업데이트 롤백 → 에러 토스트 → 재시도 버튼 |
| 리더보드 Realtime 연결 끊김 | 조용히 재연결 시도 (3회) → 실패 시 "실시간 업데이트 일시 중단" 배너 |

### 3.2 빈 시그널

| 상황 | 기대 동작 |
|---|---|
| 오늘 시그널 0개 (Edge Function 실패 등) | 빈 상태 UI: "오늘의 시그널을 준비 중이에요. 잠시 후 다시 확인해주세요." |
| 인사이트 카드 0개 | 빈 상태 UI: "인사이트를 분석하고 있어요." |
| 리더보드 데이터 없음 | "아직 참여자가 없어요. 첫 번째로 예측에 참여해보세요!" |

### 3.3 동점 처리

| 상황 | 기대 동작 |
|---|---|
| 리더보드 동점자 발생 | 정확도 동점 시 총 점수 높은 순 → 동일 시 먼저 제출한 유저 우선 |
| 동점자 UI | 같은 순위 번호 표시 (예: 공동 3위 → #3, #3, #5) |

### 3.4 마감 시간 경계

| 상황 | 기대 동작 |
|---|---|
| 마감 1분 전 예측 제출 | 정상 처리 |
| 마감 시간 정각 | 서버에서 마감 처리 → 클라이언트 슬라이더 비활성화 |
| 마감 후 예측 시도 | "예측이 마감되었습니다" 안내 → 제출 버튼 disabled |

## 5. 보안 테스트

### XSS (Cross-Site Scripting)

- [ ] 닉네임 입력 필드에 `<script>alert(1)</script>` 입력 → 스크립트 실행 안 됨 확인
- [ ] AI 생성 인사이트 카드 본문에 HTML 태그 렌더링 안 됨 확인 (`dangerouslySetInnerHTML` 미사용)
- [ ] URL 파라미터를 DOM에 직접 삽입하지 않음 확인

### CSRF

- Supabase Auth는 httpOnly 쿠키 기반 → SameSite=Lax 설정 확인
- 상태 변경 요청(예측 제출)은 인증 토큰 필수 → 미인증 요청 401 반환 확인

### 인젝션

- [ ] 닉네임, 검색어 등 사용자 입력이 SQL에 직접 삽입되지 않음 (Supabase Parameterized Query 확인)
- [ ] Edge Function 내 OpenAI 프롬프트에 사용자 입력 삽입 시 Prompt Injection 방지 — 입력 sanitize 확인

### 인증/인가

- [ ] 로그인 없이 예측 제출 API 호출 → 401 반환 확인
- [ ] 타 사용자의 예측 데이터 조회 불가 확인 (RLS 정책 검증)
- [ ] 만료된 JWT로 API 호출 → 401 반환 확인

## 6. 테스트 환경

### 도구 목록

| 도구 | 버전 | 용도 |
|---|---|---|
| Vitest | ^2.0 | 유닛/통합 테스트 |
| @testing-library/react | ^16.0 | React 컴포넌트 테스트 |
| Playwright | ^1.44 | E2E 테스트 |
| MSW (Mock Service Worker) | ^2.0 | API 모킹 |
| @vitest/coverage-v8 | ^2.0 | 커버리지 리포트 |

### 환경 설정

```
로컬 개발: Supabase 로컬 인스턴스 (`supabase start`) + MSW 모킹
CI (GitHub Actions): MSW 모킹 전용 (실제 Supabase 연결 없음)
E2E (Playwright): Vercel Preview URL 또는 로컬 Next.js 서버
```

### 테스트 실행 명령어

```bash
# 유닛/통합 테스트
pnpm test

# 테스트 + 커버리지
pnpm test:coverage

# E2E (로컬)
pnpm e2e

# E2E (CI)
pnpm e2e:ci
```
