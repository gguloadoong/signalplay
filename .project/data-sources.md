## 1. 필요 데이터 소스

### 1.1 금융 뉴스

시그널 생성 및 인사이트 카드의 핵심 원재료. 한국 시장 관련 뉴스와 글로벌 금융 뉴스를 모두 커버해야 한다.

**필요 데이터**: 제목, 본문 요약, 원문 URL, 발행 시간, 출처명, 카테고리(경제/시장/기업)

### 1.2 시장 데이터 (최소한)

예측 결과 검증을 위한 가격 데이터. MVP에서는 지수(코스피, S&P500, 나스닥) + 환율(USD/KRW) + 원자재(금, 유가)만 필요.

**필요 데이터**: 일간 종가, 등락률, 거래량

### 1.3 논문/리서치

AI 인사이트의 깊이를 높이기 위한 학술 데이터. 금융·경제 분야 최신 논문의 abstract 활용.

**필요 데이터**: 제목, abstract, 저자, 발행일, PDF 링크

### 1.4 규제/정책

중앙은행 결정, 정부 정책, 규제 변경 등 시장 임팩트가 큰 이벤트.

**필요 데이터**: 공식 발표 제목, 요약, 발행 기관, 날짜

## 2. 후보 비교

### 2.1 뉴스 API

| 항목 | NewsAPI.org | 네이버 뉴스 검색 API | Finnhub News |
|---|---|---|---|
| **커버리지** | 글로벌 영문 뉴스 80,000+ 소스 | 한국어 뉴스 특화 | 글로벌 금융 뉴스 특화 |
| **한국어 지원** | 제한적 | 완벽 | 없음 (영문 전용) |
| **무료 한도** | 100 req/day (개발), 1,000/day (표준) | 25,000 req/day | 60 req/min |
| **가격** | $449/월 (표준) | 무료 | 무료 (기본) / $49/월 |
| **응답 속도** | 빠름 | 빠름 | 빠름 |
| **금융 특화** | 없음 (범용) | 없음 (범용) | 있음 |
| **단점** | 한국어 약함, 비용 높음 | 본문 미제공 (링크만) | 한국 시장 약함 |

**결정**: MVP에서 **네이버 뉴스 검색 API**(한국어) + **Finnhub News**(글로벌 금융) 조합 사용.

### 2.2 시장 데이터 API

| 항목 | Alpha Vantage | Yahoo Finance (yfinance) | FRED API |
|---|---|---|---|

| **무료 한도** | 25 req/day (무료), 500 req/min (유료) | 비공식 — 불안정 | 무제한 (무료) |

**결정**: **Alpha Vantage**(주가/지수) + **FRED API**(미국 거시경제 보조) 사용. Yahoo Finance는 TOS 이슈로 배제.

## 4. 법적 고려

### API 이용 약관

| 서비스 | 상업적 사용 | 재배포 허용 | 주의 사항 |
|---|---|---|---|
| 네이버 뉴스 API | 조건부 허용 | 제한 — 원문 링크 필수 | 뉴스 본문 저장 금지, 링크 제공만 |
| Finnhub | 무료 플랜: 비상업, 유료 플랜: 상업 허용 | 제한 | 상업 서비스 시 유료 플랜 필수 |
| Alpha Vantage | 유료 플랜: 상업 허용 | 제한 | 데이터 재판매 금지 |
| FRED API | 허용 | 허용 (출처 표기) | 저작권 자유, 출처 명시 권장 |
| arXiv API | 허용 | 허용 | 저자/제목 출처 표기 필수 |

### 금융 데이터 재배포 규정

- **원칙**: 시장 데이터(주가, 지수)를 실시간으로 제3자에게 제공하는 것은 KRX, NYSE 등 거래소의 별도 라이선스 필요.
- **MVP 전략**: 일간 종가(EOD data)만 사용. 실시간 시세 표시는 MVP 제외 (OUT 목록 참조).
- **면책 문구**: 모든 데이터 표시 화면에 "데이터는 지연될 수 있으며 참고용입니다" 명시.

## 5. 품질 확보

### 중복 제거

- `news` 테이블의 `url` 컬럼에 UNIQUE 제약 → DB 레벨에서 중복 차단
- 유사 기사 감지: 제목 기반 fuzzy matching (Levenshtein distance < 10%) → 동일 이벤트 중복 인사이트 방지

### 신뢰도 점수

Edge Function에서 뉴스 수집 시 OpenAI를 통해 임팩트 점수(`impact_score: 1~10`) 산정:

```
프롬프트: "다음 뉴스가 한국 주식 시장에 미치는 영향을 1(미미)~10(매우 큼)으로 평가하고,
영향을 받는 섹터를 최대 3개 나열하세요."
```

- 점수 7 이상: Breaking News 후보
- 점수 5 이상: 인사이트 카드 생성 대상
- 점수 4 이하: DB 저장만, 카드 생성 제외 (비용 절감)

### 출처 신뢰도 화이트리스트

신뢰할 수 있는 출처를 코드로 관리:

```typescript
const TRUSTED_SOURCES = [
  '한국경제', '매일경제', '서울경제',
  'Bloomberg', 'Reuters', 'FT', 'WSJ',
  '금융위원회', '한국은행', 'Federal Reserve',
]
```

화이트리스트 출처는 임팩트 점수 +1 보너스 적용.

## 6. 개인정보 보호

### 수집 원칙

- 뉴스/논문/시장 데이터는 **개인정보 비포함** — 별도 GDPR/개인정보보호법 제약 없음
- 사용자 데이터(예측, 점수)는 Supabase 내부에만 보관

### 로그 정책

- Edge Function 로그에 사용자 ID, IP 등 개인정보 기록 금지
- 뉴스 수집 로그는 URL + 타임스탬프만 기록

### 데이터 보존

| 데이터 | 보존 기간 | 삭제 방법 |
|---|---|---|
| `news` 테이블 | 90일 | cron으로 자동 삭제 |
| `signals` 테이블 | 1년 | 수동 아카이브 |
| `predictions` 테이블 | 영구 (익명화 후) | 탈퇴 시 user_id null 처리 |
| `insights` 테이블 | 180일 | cron으로 자동 삭제 |
