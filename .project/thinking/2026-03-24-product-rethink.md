# 서비스 근본 재검토 — 2026-03-24

## 발단

CEO 질문: "이 앱의 키 피쳐가 뭐야? 사용자들이 뭘하길 바라는거야? 들어올 이유가 있어?"

## 문제 인식

### 현재 앱이 제공하는 것
- AI가 경제 뉴스 3개를 요약해줌
- 호재/악재/영향없음 예측 게임
- 리더보드 랭킹
- AI 강세vs약세 토론 피드

### 사용자가 실제로 필요한 것
- "오늘 시장에 무슨 일 있었어? 놓친 거 없어?" → 정보 FOMO
- "내가 들고 있는 종목 괜찮은 거야?" → 투자 불안
- "이 뉴스가 내 투자에 어떤 영향이야?" → 해석 능력 부족
- "앞으로 어떻게 될까?" → 전망에 대한 갈증

### 갭 분석
| 해결해야 할 문제 | 우리가 만든 것 | 갭 |
|----------------|-------------|-----|
| "오늘 뭐 놓쳤지?" | 뜬구름 매크로 시그널 3개 | 유저 관심사와 무관 |
| "내 종목 괜찮아?" | 유저의 관심사와 무관한 뉴스 | 개인화 제로 |
| "이게 내 투자에 어떤 의미야?" | 호재/악재 3지선다 게임 | 해석이 아니라 퀴즈 |
| "앞으로 어떻게 될까?" | AI 토론 (근거 없는 양비론) | 실질적 전망 없음 |
| 토스증권 종목과 연동 | 연동 없음 | 토스 안에 있으면서 토스 데이터 미활용 |

## 근본 실수

**"투자 불안을 해소하는 도구"를 만들어야 했는데, "투자를 소재로 한 퀴즈 게임"을 만들었다.**

게임이 목적이 아니라 습관을 만드는 포장이어야 했다. 포장이 본체가 됐다.

## 더 근본적인 실수

프롬프트를 받자마자 "뭘 만들지"부터 시작했다.

```
실제로 한 것:    아이디어 → 기술 스택 → 코드 → PR → 배포
해야 했던 것:    아이디어 → "이게 진짜 문제야?" → "누가 왜 쓰지?" → "기존에 왜 안 되지?" → 그래서 뭘 만들지
```

킥오프에서 에이전트 6명이 발표했지만, 아무도 "이거 사람들이 쓸까?"를 질문하지 않았다.
12개 PR, 39개 테스트, v4 프로세스까지 만들었지만 — 아무도 원하지 않는 것을 완벽한 프로세스로 만든 것.

## 원래 아이디어의 진짜 인사이트

CEO 원문: "사람들의 가려운 부분을 긁어주는 게 목적"

"가려운 부분" = 투자자들이 매일 느끼는 불안과 FOMO
- 사람들이 하루에 토스증권을 5번 여는 이유는 매매하려는 게 아니라 불안해서
- "놓치면 안 되는 것"에 대한 강박
- 정보는 넘치는데 "나한테 의미 있는 것"을 걸러주는 사람이 없음

이 인사이트는 유효하다. 실행이 잘못됐을 뿐.

## 핵심 질문 (답변 필요)

이 프로젝트를 계속할지 결정하기 전에 답해야 하는 질문들:

1. **"30초에 오늘 내가 알아야 할 투자 정보를 파악하는 것"** — 이게 우리가 제공하려는 가치가 맞는가?
2. **개인화가 가능한가?** — 앱인토스에서 토스증권 보유 종목 데이터에 접근할 수 있는가? 아니면 유저가 직접 관심 종목을 선택하는 방식인가?
3. **게임은 포장인가 본체인가?** — 습관 형성을 위한 게이미피케이션인가, 아니면 게임 자체가 서비스인가?
4. **경쟁 우위가 있는가?** — 토스증권 뉴스탭, 네이버 금융, 알파스퀘어 대비 우리만의 차별점은?
5. **앱인토스가 맞는 플랫폼인가?** — 이 서비스의 본질이 "토스 안에서만" 의미가 있는 것인가?

- 장점: 깨끗한 출발. 단점: 시간

- 다른 아이디어로 전환

## 교훈 (v4 프롬프트에 반영 필요)

- "기존 대안이 이 문제를 해결하지 못하는 이유는?"
- "사용자가 이 앱을 열어야 하는 이유를 한 문장으로 말할 수 있는가?"
- 이 질문에 팀 전원이 합의하기 전까지 코드를 작성하지 않는다.
